    Claude Quick Reference¶

    Getting Started¶

    Claude Code is an AI-powered CLI assistant for software development tasks.

    Bash
    # Start Claude Code
claude

# Get help
/help

# Clear conversation
/clear

    Common Workflows¶

    File Operations¶

    Bash
    # Read a file
"Read the file at path/to/file.py"

# Edit specific parts
"Change the function name from oldName to newName in file.py"

# Create new files (only when necessary)
"Create a new config.json file with these settings..."

    Code Tasks¶

    Bash
    # Debug
"Find and fix the bug in src/main.py"

# Refactor
"Refactor the UserService class to use dependency injection"

# Add features
"Add error handling to the API endpoints"

# Explain code
"Explain what the calculate_metrics function does"

    Git Operations¶

    Bash
    # Check status
"Run git status and show me what's changed"

# Create commit
"Commit these changes with an appropriate message"

# Create PR
"Create a pull request for this feature"

    Tips for Better Results¶

    Be Specific¶

    Bash
    # Vague
"Fix the code"

# Better
"Fix the TypeError in utils.py line 42 when parsing dates"

    Provide Context¶

    Bash
    # Include relevant details
"Update the API to use JWT auth. We're using Express and jsonwebtoken library"

    Break Down Complex Tasks¶

    Bash
    # Instead of: "Build a complete user authentication system"
# Try:
1. "Create user model with email and password fields"
2. "Add password hashing with bcrypt"
3. "Create login endpoint with JWT tokens"

    Quick Commands Reference¶

    Task Example Command
    Search code "Find all files that import pandas"
    Run tests "Run pytest and show me the results"
    Check types "Run the TypeScript compiler and fix errors"
    Format code "Format this file with black"
    Install deps "Install the required packages from requirements.txt"
    Create branch "Create a new branch called feature/user-auth"

    Remember¶

    • Claude works best with specific, clear requests
    • You can run multiple commands in parallel when they're independent
    • Always review changes before committing
    • Use the todo list for complex multi-step tasks
    • Claude has access to web search for up-to-date information

    Working on This Site¶

    IMPORTANT WORKFLOW: After every change to markdown files:

    1. Commit and push changes

      Bash
      git add <changed files>
git commit -m "Descriptive message"
git push

    2. Kill and restart local server

      Bash
      pkill -f "mkdocs serve"
cd "/Users/aaronscher/Desktop/1_Projects/aaron-scher github website/home"
mkdocs serve

    3. View fresh render at: http://127.0.0.1:8000/home/

    This ensures I can see the changes rendered properly in my browser with all MathJax equations displaying correctly.

    About This Site¶

    This is a personal knowledge base where I document concepts, formulas, and workflows to help my future self quickly get back up to speed.

    Purpose¶

    I have a tendency to forget things, so I write detailed notes in tutorial form. This helps me:

    • Quickly refresh my memory on topics I've studied
    • Maintain a searchable reference for formulas and concepts
    • Build a structured knowledge base over time

    How It Works¶

    Built with MkDocs Material and deployed via GitHub Pages. Math equations are rendered with MathJax.

    MathJax/LaTeX Rendering Requirements¶

    IMPORTANT: For proper math rendering with MkDocs + pymdownx.arithmatex + MathJax:

    Display Math (standalone equations)¶

    Markdown
    # CORRECT - $$ on separate lines with blank line before
Text before the equation.

$$
E = mc^2
$$

More text after.

# WRONG - inline with text
Text $$E = mc^2$$ more text

    Inline Math (within text)¶

    Markdown
    # Use single $ for inline math
The energy is $E = mc^2$ in this system.

    MathJax Configuration¶

    The docs/javascripts/mathjax.js file must be configured to accept the delimiter styles:

    JavaScript
    window.MathJax = {
  tex: {
    inlineMath: [["\\(", "\\)"], ["$", "$"]],      // Support both styles
    displayMath: [["\\[", "\\]"], ["$$", "$$"]],   // Support both styles
    processEscapes: true,
    processEnvironments: true,
    tags: 'ams'
  },
  options: {
    ignoreHtmlClass: ".*|",
    processHtmlClass: "arithmatex"
  }
};

    Common Issues¶

    • Backslashes showing: MathJax delimiter mismatch - check mathjax.js config
    • Equations not rendering: Missing blank lines before $$ blocks
    • Changes not appearing:
    • Kill mkdocs server: pkill -f "mkdocs serve"
    • Restart: mkdocs serve
    • Hard refresh browser (Cmd+Shift+R)
    • Check http://127.0.0.1:8000/home/ (note the port and path)
